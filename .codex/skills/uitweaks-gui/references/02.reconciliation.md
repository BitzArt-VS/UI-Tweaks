# Reconciliation

`BuildRenderTree` never instantiates children — it writes slot declarations (`Add<T>(key)`) into the builder. The builder owns all instances.

**Pass phases:**
1. Blueprint — `BuildRenderTree` runs, fills frame buffer.
2. Diff — each frame matched against slot map by `(ComponentType, key)`.
3. Patch — reused: layout reset, own-slot config applied, external config applied, lifecycle called. New: instance created, `OnInitialized` called. Removed: `Dispose` called, slot pruned.

**Lifecycle order per component per pass:**
1. `LayoutParameters` reset to framework defaults for layout components.
2. `ConfigureSlot(IGuiSlotBuilder)` — every pass for `GuiNode` subclasses; component-owned layout defaults and callbacks are registered here.
3. External configuration actions applied (`Configure`, `ConfigureLayout`, inline layout arguments).
4. Own-slot and external callbacks composed; own-slot callbacks run before external callbacks for the same event.
5. `OnInitialized()` — first mount only.
6. `OnParametersSet()` — every pass (after `OnInitialized` on first pass).
7. Children reconciled recursively.

**Rules:**
- Slot identity = `(ComponentType, key)`. Must be unique among siblings.
- Type mismatch at existing key → dispose old, create new.
- `BuildRenderTree` is pure declaration only.

**Scoped rebuilds:** `RequestReconcile()` schedules a rebuild of that component's subtree only. Renderer batches via swap-buffer — zero allocation, safe against re-entrant `Schedule()` calls during `Run()`. When a parent reconcile recursively rebuilds a child subtree, it `Cancel`s any separately scheduled rebuild for that child fragment — preventing redundant work when parent and descendants both call `RequestReconcile` in the same frame.

**Drain order caveat:** the active drain pops entries in arbitrary order. If parent and child both schedule rebuilds in the same frame and drain pops the child first, the child's subtree is rebuilt once before the parent is popped; the parent's rebuild then re-covers (and cancels remaining descendants of) that subtree. Worst-case overhead is one redundant subtree rebuild per ancestor in the cascade. This is a rare scenario in practice — typical updates involve a single component's `RequestReconcile`.
