# Layout Parameters

`GuiComponentLayoutParameters` — mutated by config actions during blueprint phase, consumed by layout pass.

| Property | Type | Default | Notes |
|---|---|---|---|
| `Margin` | `GuiThickness` | Zero | Outside border — separates from siblings/parent edge |
| `Padding` | `GuiThickness` | Zero | Inside border — separates border from content/children |
| `Width` | `GuiSize` | Auto | Explicit override; overrides `WidthMode` when resolvable |
| `Height` | `GuiSize` | Auto | Explicit override; overrides `HeightMode` when resolvable |
| `WidthMode` | `GuiSizeMode` | **FitContent** | Applied when `Width` is Auto |
| `HeightMode` | `GuiSizeMode` | **FitContent** | Applied when `Height` is Auto |
| `Positioning` | `GuiComponentPositioning` | Relative | Relative = stack flow; Absolute = independent placement |
| `Direction` | `GuiDirection` | Vertical | How *this* component stacks its own children |
| `HorizontalAlignment` | `GuiHorizontalAlignment` | **Left** | Cross-axis alignment within parent (vertical stacks) / both axes (absolute) |
| `VerticalAlignment` | `GuiVerticalAlignment` | **Top** | Cross-axis alignment within parent (horizontal stacks) / both axes (absolute) |

**`GuiThickness`** — immutable value type `(Top, Right, Bottom, Left)`. Ctors: uniform, vertical+horizontal, four-sided. Helpers: `.Horizontal`, `.Vertical`.

**`GuiSize`** — immutable value type used by `Width` and `Height`. Default/`Auto` means "defer to `GuiSizeMode`"; implicit `int`/`double` conversion creates fixed logical-pixel sizing; implicit `string` conversion parses fixed numbers and percentages like `"50%"`; `GuiSize.Fraction(fraction, minimum?, maximum?)` resolves against the current arrange-time available extent and then clamps.

**`GuiMeasuredSize`** — immutable value type `(Width, Height)` returned by `Measure` and `MeasureChildren`.

**`GuiComponentBounds`** — immutable value type `(X, Y, Width, Height)`. Computed by layout pass, forwarded to `Render`. Helpers: `.Right`, `.Bottom`.

## `GuiSizeMode`

| Value | Behaviour |
|---|---|
| `Fill` | Resolves to `(availW, availH)` directly — always fills the available space. `Measure()` is **not** called. Note: `Fill = 0` is the C# enum default, but `GuiComponentLayoutParameters` explicitly defaults both `WidthMode` and `HeightMode` to `FitContent`. |
| `FitContent` (framework default) | `max(MeasureChildren(...), Measure(...))` per dimension, then adds this component's own `Padding`. Leaf components override `Measure()` to report intrinsic size; containers rely on children. |

Explicit fixed `Width`/`Height` always wins. Fractional constraints also win on bounded axes; on an unbounded scroll-measure axis they fall back to the mode path, same as `Fill`.

## Size resolution (`ResolveSize`)

Priority per dimension:
1. Resolvable `Width`/`Height` constraint → fixed value or fraction of current available extent.
2. `FitContent` → `max(childBuilder.MeasureChildren(...), instance.Measure(...))` + padding. Both paths use content-available space (after subtracting padding).
3. `Fill` → `(availW, availH)` directly. `Measure()` not called.

Both `Render` and `MeasureChildren` share the same `ResolveSize` helper to ensure consistent sizing at every tree level.

## Layout pass (`GuiRenderTreeBuilder.Render`)

Called each frame after reconciliation. For every slot in render order:

1. **ResolveSize** — runs the priority chain above. Available space subtracts the slot's own margin first.
2. **Position** — `Relative`: place at cursor + margin, advance cursor along `direction`. `Absolute`: pin to content-area origin + margin, cursor unchanged.
3. **Align** — apply cross-axis alignment offset (relative children) or both-axis alignment (absolute children). Slack = `availableExtent − slotExtent`; `Left`/`Top` (defaults) = 0, `Center` = slack/2, `Right`/`Bottom` = slack. No-op when slack ≤ 0 (e.g. `Fill` mode).
4. **Render** — `slot.Instance.Render(context, allocated)`.
5. **Recurse** — inset by slot `Padding` → child content → `slot.ChildBuilder.Render(context, childContent, lp.Direction)`.
6. **RenderOverlay** — `slot.Instance.RenderOverlay(context, allocated)` after children draw, before the next sibling slot begins.

Each slot's full subtree paints as a unit (background → children → overlay) before the next sibling starts. Sibling overlays do **not** form a separate top pass — a later overlapping sibling can paint over an earlier sibling's overlay.

## `MeasureChildren`

`GuiRenderTreeBuilder.MeasureChildren(availW, availH, direction) → (Width, Height)`

Iterates relative children, resolves each slot's size, then:
- `Vertical`: `totalW = max(slotW + margin.H)`, `totalH = Σ(slotH + margin.V)`
- `Horizontal`: `totalW = Σ(slotW + margin.H)`, `totalH = max(slotH + margin.V)`

Absolute children are excluded from the sum.

## `Measure` contract

`IGuiComponent.Measure(double availableWidth, double availableHeight) → (double Width, double Height)`

Called by `FitContent` resolution alongside child measurement; the per-dimension max of the two is used. Receives content-available dimensions (available space minus padding). Default returns `(0, 0)` — no intrinsic size. Override in leaf components (text, icons) to report natural size. Not called for `Fill`.

## Configuring layout

Layout parameters are set inline as named arguments on the slot-declaration extension methods (`Add<T>`, `AddContainer`, `AddLabel`):

| Argument | Effect |
|---|---|
| `width: x` / `height: y` | Sets explicit `Width` / `Height`; numeric values become fixed sizes, strings like `"50%"` become fractional sizes, `GuiSize.Fraction(...)` allows min/max clamps |
| `widthMode: m` / `heightMode: m` | Sets `WidthMode` / `HeightMode` |
| `fill: true` | Sets both modes to `Fill` (overrides any individual mode arg via `+=`) |
| `margin: t` / `padding: t` | Sets `Margin` / `Padding` |
| `direction: d` | Sets the slot's child stacking `Direction` |
| `positioning: p` | Sets `Positioning` (Relative / Absolute) |
| `horizontalAlignment: a` | Sets `HorizontalAlignment` (Left / Center / Right) |
| `verticalAlignment: a` | Sets `VerticalAlignment` (Top / Center / Bottom) |

For anything else, chain `.Configure(c => ...)` (component state) or `.ConfigureLayout(lp => ...)` (layout state). There are currently **no** dedicated fluent shorthands like `.Fill()` or `.FitContent()` — only the inline named arguments above and `Configure`/`ConfigureLayout`.

## Alignment

`HorizontalAlignment` (`Left` / `Center` / `Right`) and `VerticalAlignment` (`Top` / `Center` / `Bottom`) position a slot within slack space — the difference between the parent's available extent and the slot's resolved size. `Left`/`Top` are the defaults and reproduce pre-alignment behaviour.

When alignment applies:

| Slot kind | Parent direction | `HorizontalAlignment` | `VerticalAlignment` |
|---|---|---|---|
| Relative | Vertical | Cross axis ✔ | No-op (flow axis) |
| Relative | Horizontal | No-op (flow axis) | Cross axis ✔ |
| Absolute | (any) | ✔ | ✔ |

A slot only has slack to align against when its size is *less than* the parent-provided extent on that axis — i.e. resolved via `FitContent` or an explicit `Width` / `Height` smaller than what the parent offers. `Fill` consumes the full extent, so alignment becomes a no-op naturally; alignment never enlarges a slot.

Layout-transparent wrappers (pure `IGuiNode` slots) have no `LayoutParameters` and therefore no alignment of their own — set alignment on the children declared inside the wrapper instead.

## Layout-transparent wrappers

The framework's node hierarchy splits into two interfaces:

- `IGuiNode` (public, lightweight) — the minimum tree-participation contract: `RenderFragment`, `Attach`, lifecycle hooks, `Render` / `RenderOverlay`. **No layout members.**
- `IGuiComponent : IGuiNode` (public) — adds `LayoutParameters` and `Measure`. Slots whose instance implements this participate in layout normally.

A node is **layout-transparent** simply by *not* implementing `IGuiComponent` — i.e. by inheriting from `GuiNode` (or implementing only `IGuiNode` directly) instead of `GuiComponent`. There is no marker interface; the type system encodes the capability. The layout pass dispatches via `slot.Instance is IGuiComponent layoutComp`:

- For layout-participating slots: the existing path runs (`LayoutParameters`, `Measure`, allocated bounds, child padding inset).
- For pure `IGuiNode` slots:
  - The layout pass does not allocate space for the slot itself.
  - The wrapper has no `LayoutParameters` to consult.
  - The wrapper's child builder is inlined at the parent's level — its slots flow at the parent's declaration site as if they had been declared directly there, using the parent's `direction` / `contentBounds`.
  - The wrapper's `Render` / `RenderOverlay` hooks still run, once per pass, with bounds spanning the **union of its inner children's allocated rectangles** along the parent's flow axis. Wrappers that need to know the area their content occupies (e.g. `GuiTooltip` registering its hover region) read those bounds.
  - Mouse-handler regions registered via the slot's own builder use the same union bounds.

`GuiRenderTreeBuilder.Render` and `MeasureChildren` recurse into transparent slots' child builders via internal `RenderInto` / `AccumulateMeasure` helpers that operate on externally-managed cursors / accumulators — no allocation, no tree restructuring.

Use `GuiNode` for cross-cutting wrapper concerns that must not perturb layout: tooltips, hover-tracking decorators, debug overlays, future portal-style components. Combine with cascading values to publish state for descendants while staying invisible to layout. Current implementor: `GuiTooltip`.

Constraints: transparent slots are always treated as relative children (a transparent wrapper has no `Positioning` knob to begin with); absolute positioning should be declared on the children inside the wrapper instead.

API constraints:
- `IGuiRenderTreeBuilder.AddComponent<T>` is constrained to `T : IGuiNode, new()` so both kinds of slot can be declared through the same builder.
- Public declarations use `Add<T>` for both layout-participating `IGuiComponent` slots and layout-transparent `IGuiNode` slots. Layout arguments are valid only for component slots.
- `IGuiComponentBuilder<T>` is constrained to `T : IGuiNode` and extends `IGuiSlotBuilder`.
- `Configure<T>` is constrained to `T : IGuiNode`; `OnMouse*` / `OnKey*` extensions target `IGuiSlotBuilder` so the same callback API works for external slots and `ConfigureSlot`.
- `ConfigureLayout<T>` is the sole extension constrained to `T : IGuiComponent` — there's no layout to configure on a pure node.
